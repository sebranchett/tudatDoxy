17 // Some improvements are foreseen:
18 // * The application of legs departure vs swingby vs capture is insecure. This could be improved.
19 // * The minimum pericenter radii are not necessary for the velocity formulation model. The vector
20 // that contains these radii does contain a value for it though. This could be improved.
21 // * The checks that are performed are not sufficient and by inputting incomplete information,
22 // the function may crash/behave strangely.
23 // * Variables could be passed alternatively by data containers of some kind.
24 // * When returning the maneuvers, no check is present to check if the trajectory has been
25 // calculated already. This could be improved.
26 // * Default constructor may be regarded as unsafe. Other implementations may be possible.
