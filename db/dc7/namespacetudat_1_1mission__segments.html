Calculates the deltaV required to perform a certain gravity assist. This function essentially tries to patch the incoming and outgoing velocity using an unpowered gravity assist. If however the required bending angle cannot be obtained, the deltaV required to patch this is calculated. Likewise the deltaV required to patch the hyperbolic excess velocities is calculated. The sum of the two is the total deltaV required to perform the maneuver and is returned.

Parameters
centralBodyGravitationalParameterGravitational parameter of the swing-by body.[m^3 s^-2]
centralBodyVelocityHeliocentric velocity of the swing-by body. [m s^-1]
incomingVelocityHeliocentric velocity of the spacecraft before the swing-by. [m s^-1]
outgoingVelocityHeliocentric velocity of the spacecraft after the swing-by. [m s^-1]
smallestPeriapsisDistanceClosest allowable distance to the swing-by body. [m]
useEccentricityInsteadOfPericenterFlag to indicate the iteration procedure for matching the bending angle. [-]
speedToleranceTolerance at which the velocity effect deltaV is deemed 0.0. [-]
rootFinderShared-pointer to the rootfinder that is to be used. Default is Newton-Raphson using 1000 iterations as maximum and 1.0e-12 relative X-tolerance.
Returns

Solves the Lambert Problem using Izzo's algorithm. This code is an implementation of the method developed by Dario Izzo from ESA/ACT and publicly available at: http://keptoolbox.sourceforge.net/ After verification and validation, it was proven that this algorithm is faster and more robust than the implemented Lancaster & Blanchard and Gooding method. Notably, this method does not suffer from the near-pi singularity (pi-transfers are by nature singular). This method works in adimensional units, meaning that position, time-of-flight and gravitational parameter can be provided in any units, as long as they are coherent across all quantities. Results will be returned in the same units as the input variables. Note that while this implementation does not support multi-revolution transfers, the original algorithm (see link above) does. The root-finder (Secant Method) is currently hard-coded.

cartesianPositionAtArrivalCartesian position at arrival. [Input]

gravitationalParameterGravitational parameter of the central body. [Input]
cartesianVelocityAtDepartureVelocity at departure. [Output]
cartesianVelocityAtArrivalVelocity at arrival. [Output]
isRetrogradeBoolean flag to indicate direction of motion. [Input, Optional]
convergenceToleranceConvergence tolerance for the root-finding process. [Input, Optional]
maximumNumberOfIterationsMaximum number of iterations of the root-finding process. [Input, Optional]

Solves the Lambert Problem using Lancaster and Blanchard's algorithm with further improvements by Gooding. The number of revolutions from departure to arrival body is zero by definition in this routine. This can be made user-defined later on. The resulting trajectories are in anti-clockwise (prograde) direction.
